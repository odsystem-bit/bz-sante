# Politique de sécurité

La sécurité de BZ Santé est une priorité absolue compte tenu de la sensibilité des données de santé. Si vous découvrez une vulnérabilité, merci de suivre la procédure ci-dessous.

## Versions supportées

| Version | Supportée |
| :--- | :--- |
| 0.1.x | :white_check_mark: Oui |

## Signaler une vulnérabilité

1. **Ne publiez pas** la vulnérabilité publiquement (issue, PR, forum, etc.).
2. Envoyez un e-mail à l'équipe de sécurité : [À compléter](mailto:)
3. Incluez dans votre message :
   - La description du problème
   - Les étapes pour reproduire
   - L'impact potentiel
   - Les suggestions de correction si vous en avez

## Engagement

- Nous accusons réception de votre signalement dans les 5 jours ouvrés.
- Nous vous tenons informé de l'avancement de la correction.
- Nous créditons publiquement les chercheurs qui signalent des vulnérabilités de manière responsable, sauf opposition de leur part.

## Bonnes pratiques de sécurité

- Authentification forte par rôle
- Chiffrement des données de santé en transit et au repos
- Journalisation et surveillance des accès
- Anonymisation des données statistiques
- Conformité aux réglementations de protection des données de santé
- Audits réguliers et tests de pénétration

Merci de contribuer à la sécurité de BZ Santé.
